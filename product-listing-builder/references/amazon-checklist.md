# Amazon checklist

Use this reference when the target marketplace includes Amazon.

## Core principle

Write for scan speed, factual clarity, and clean search intent. Remove unsupported claims and keep the variant shown in the images aligned with the text.

## Title checks

- Lead with brand, product type, and the most important evidence-backed differentiators.
- Include only useful attributes such as size, color, quantity, material, or compatibility when they matter for selection.
- Remove emojis, duplicate keyword strings, all-caps emphasis, and vague promo phrases.
- Avoid stuffing the title with every possible search term.

## Bullet checks

- Start with the strongest buyer-relevant benefit.
- Pair each feature with why it matters.
- Cover use case, key material/build details, sizing or compatibility, included items, and care or constraint notes when relevant.
- Avoid repeating the title or writing five versions of the same claim.

## Description checks

- Expand the product story without drifting into filler.
- Clarify fit, usage context, limitations, and what is included.
- Keep wording easy to scan even in paragraph form.

## Search-term checks

- Add synonyms, abbreviations, intent phrases, and relevant attribute combinations.
- Avoid obvious duplication of the exact title string.
- Exclude competitor names and unsupported claims.

## Image checks

- Confirm the hero image clearly matches the sold variant.
- Look for coverage across angle, detail, texture/material, scale, included items, packaging, and usage context where relevant.
- Flag when a key differentiator exists in text but not in images.

## Variant checks

- Group only true variants of the same base product.
- Name the variant axis explicitly.
- Split unrelated products even when the brand overlaps.

## Escalate when needed

- If exact category limits, restricted-product rules, or current publishing constraints matter, verify the latest official Amazon documentation before claiming full compliance.
